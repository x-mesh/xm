# xm 케이스 스터디

xm이 실제로 무엇을 잡아내는지, **만들어낸 예시가 아니라 실제 세션의 아티팩트**로 보여준다.
세 케이스 모두 같은 날(2026-07-04~05) 하나의 실제 작업 흐름에서 나왔고, 모든 주장은
저장소 안의 파일 경로로 추적 가능하다.

| # | 케이스 | 한 줄 요약 | 주제 |
|---|--------|-----------|------|
| 1 | [적대 패널이 자기 벤더의 버그를 잡다](01-cross-vendor-catches-own-bug.md) | 5-벤더 브레인스토밍에서 claude 레인만 무효 출력 → 8실험 격리 → 근본 원인(컨텍스트 오염) 수정 | 크로스벤더 검증 |
| 2 | [자기채점을 믿지 마라](02-eval-loop-catches-real-error.md) | 자기채점 8.1 → 3-judge 패널 6.99 FAIL(3/3이 같은 실오류 독립 적발) → 수정 → 7.82 PASS | 측정 기반 신뢰 |
| 3 | [메모리가 결정을 바꾸다](03-memory-changes-decisions.md) | 5주 전 결론을 오늘 transcript 실측이 반증 → 설계 전환; 5일 전 전략 문서를 probe 에이전트가 발굴·인용 | 축적 아티팩트 재사용 |

## 정직 고지 (읽고 판단하세요)

- **N=1**: 세 케이스 모두 제작자 본인의 사용에서 나왔다. "다른 팀에도 유효한가"는 아직 미검증이며,
  이 문서의 목적이 바로 그 검증(외부 반응 수집)이다.
- **비용 수치는 추정 라벨**: 서브에이전트별 실측 토큰은 현재 하네스가 노출하지 않는다.
  본 문서의 비용 언급은 `cost_source: estimated`이며 실측이라 주장하지 않는다.
- **케이스 2의 심판은 단일 벤더(sonnet)**: 크로스벤더 심판이 아니다. 크로스벤더 검증 사례는 케이스 1.
