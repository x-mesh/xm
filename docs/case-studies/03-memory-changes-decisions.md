# 케이스 3 — 메모리가 결정을 바꾸다 (그리고 반증당한다)

**날짜**: 2026-07-04~05 · **주제**: 세션을 넘어 축적된 아티팩트가 실제 결정에 개입한 순간들

## 사례 A: 5주 전의 결론이 오늘 반증되다

2026-05-04의 프로젝트 메모리는 이렇게 결론냈다: *"SKILL.md frontmatter `model:` 필드가
공식적·결정론적·leader 포함 모델 override — 이것이 정답."* 이 결론은 5주간 설계의 전제였다.

오늘(2026-07-04) 세션 transcript를 전수 실측하자(어시스턴트 메시지 314건의 실제 런타임 모델
필드) — **frontmatter `model: opus`가 활성인 스킬 턴에서도 리더는 314/314 전부 세션 모델로
실행**되고 있었다. 전제가 현행 하네스에서 무효.

결과: 모델 라우팅을 frontmatter 고정에서 **inherit(부재=세션 모델 상속)** 로 전환하는 릴리스
(x-build@2.12.0, xm@2.4.41)가 나왔고, 낡은 메모리에는 반증 주석이 달렸다. 메모리는 "정답 보관소"가
아니라 **반증 가능한 가설 대장**으로 동작했다.

## 사례 B: 5일 전의 문서를 에이전트가 발굴해 인용하다

전략 재포지셔닝을 probe(전제 검증)할 때, 대안 탐색 에이전트가 지시 없이
`docs/strategy/xm-differentiation.md`(2026-06-29, 5모델 패널 검증본)를 저장소에서 찾아
**§8-2의 미검증 가정("전략 증명에 사용자가 비용·지연을 지불하는가")과 §7의 검정력 설계
(d≈0.5, 군당 n≈64)를 정확히 인용**해 "지금 코드를 짓지 말고 이 가정부터 검증하라"고 반박했다.
인용 문구는 원문 대조로 일치 확인됨.

## 사례 C: 심판이 메모리를 교차 참조하다 — 그리고 오독하다

재채점의 adversarial 심판은 하루 전 메모리("vendor 추상화 — Claude/Codex 이중 지원")를
근거로 "5벤더 실행이 실제였는지 의심된다"고 공격했다. **오독이었다** — 그 메모리는 비용
엔진의 단가 추상화 층 이야기고, 패널 어댑터는 별개로 5벤더를 지원한다. 원본 응답 파일 5개와
kiro 출력의 ANSI 이스케이프 146개가 반증 증거로 제시되어 기각됐다.

교훈이 양면이다: 메모리 교차 참조는 심판을 강하게 만들지만, **메모리 오독은 새로운 오류
원천**이다 — 그래서 반박은 기록으로 남기고(disposition), 메모리엔 맥락 경계를 명시한다.

## 정량 배경 (주의해서 읽을 것)

이 저장소의 실질 세션(어시스턴트 메시지 5개 이상) 61개 중 60개(98%)에서 축적 메모리의 능동
조회가 관측됐다. **단, 이 수치는 증거로 쓰기에 부적합하다**: (1) N=1 사용자, (2) 세 겹의 메모리
메커니즘(mem-mesh MCP / 하네스 네이티브 메모리 / `.xm` 아티팩트)이 동시 가동 중이라 기여
귀속이 불가능(confound), (3) 조회 일부는 규칙이 강제한 것. 위의 사례 A/B/C 같은
**결정을 바꾼 구체 사건**이 이 문서의 실제 증거이고, 98%는 참고 배경일 뿐이다.
