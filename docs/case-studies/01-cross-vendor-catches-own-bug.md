# 케이스 1 — 적대 패널이 자기 벤더의 버그를 잡다

**날짜**: 2026-07-04~05 · **소요**: 발견~수정~회귀테스트 약 1시간 · **코드 변경**: `x-panel/lib/x-panel/adapters.mjs` + 회귀 테스트 2건

## 상황

xm의 차별화 전략을 5개 벤더(claude/codex/gemini/cursor/kiro)에 동시에 브레인스토밍시켰다
(`xm panel cross`). 4개 벤더는 정상적인 반박·아이디어를 냈는데, **claude 레인만 940바이트의
무효 출력**을 반환했다 — 과제 수행 없이 자기 컨텍스트 조각을 에코.

크로스벤더 검증이 핵심 자산인 도구에서 자기 벤더가 깨져 있었던 것. 단일 벤더로만 돌렸다면
**이 버그는 관측 자체가 불가능했다** — 비교 대상이 없으므로.

## 격리 과정 (8실험 매트릭스)

| 실험 | cwd | env | 프롬프트 | 결과 |
|---|---|---|---|---|
| 1 | repo | 상속 | 짧음 | ✅ 정상 |
| 2·3 | repo | 상속 | 김(2.9KB) | ❌ 스캐폴드/사고 누출 |
| 4 | repo | **세척** | 김 | ❌ 여전히 에코 → **env 무관 확정** |
| 5 | **repo 밖** | 세척 | 김 | ✅ 10.5KB 정상 분석 → **cwd가 범인** |
| 6 | repo | 상속 | 김 + `--bare` | ❌ OAuth 소실 (기각) |
| 7 | repo | 상속 | 김 + `--setting-sources user` | ❌ user 스코프 주입 잔존 (기각) |
| 8 | repo | 상속 | 김 + `--output-format json` | **결정타**: output 55토큰 = 시스템 스캐폴드 조각 |

실험 8이 원인을 확정했다: CLI 스트림 혼선이 아니라, repo cwd에서 자식 `claude -p`가
프로젝트 CLAUDE.md + hook 주입 컨텍스트(~47K 토큰)를 조립하고, **모델이 긴 프롬프트에서
답변 대신 그 스캐폴드를 재생**한 것.

## 수정과 검증

`promptSpawnOpts()` — claude 프롬프트 실행만 중립 tmp cwd로 격리(5개 스폰 지점).
타 벤더는 의도적으로 caller cwd 유지(codex `--sandbox read-only`는 repo를 읽어야 함).

- 수정 후 동일 실패 조건 재실행: **5,455자 정상 분석, 누출 마커 0건**
- 회귀 테스트 2건 추가, 전체 테스트 통과

## 증거 사슬 (전부 저장소 안에 실존)

- 실패 원본: `.xm/cross/panel-20260704-232920-420/claude-claude-opus-4-8.json` (940B 에코)
- 정상 대조: 같은 디렉토리의 `codex.json`/`agy.json`/`cursor.json`/`kiro-*.json` (4.8~9.4KB)
- 수정: `x-panel/lib/x-panel/adapters.mjs` `promptSpawnOpts()` + 주석에 실험 결과 요약
- 회귀 테스트: `test/x-panel-cli.test.mjs` "promptSpawnOpts — claude prompt runs are cwd-isolated"

## 이것이 보여주는 것

1. **이질 벤더 비교는 단일 벤더가 볼 수 없는 고장을 드러낸다** — 4/5 정상이라는 대조군이 없었으면 "그냥 결과가 부실하네"로 넘어갔을 것.
2. **실패를 숨기지 않는 파이프라인** — 패널 결과에 4/5를 4/5로 기록했기 때문에(5/5로 위장하지 않았기 때문에) 추적이 시작됐다.

## 한계 (정직)

- 이 버그 자체가 xm의 버그다 — "우리 도구가 우리 버그를 잡았다"는 자기참조 사례.
- 근본 원인의 절반은 Claude CLI의 중첩 컨텍스트 동작이며, xm의 수정은 회피(cwd 격리)다.
