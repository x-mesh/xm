# x-probe Question Rubric

Evidence grade별 후속 질문 템플릿. SKILL.md에서 Phase 2 진행 시 참조.

## Evidence Grades

| Grade | 정의 | 예시 |
|-------|------|------|
| assumption | 검증되지 않은 믿음. 출처/날짜/측정 없음 | "사용자가 원할 것이다" |
| heuristic | 간접 근거 존재. 유사 사례, 경험, 패턴 기반 | "비슷한 도구에서 이 방식이 작동했다" |
| data-backed | 직접 측정, 구체적 수치, 명시적 출처 | "지난달 설문 결과 73%가 필요하다고 응답" |

## Grade-Specific Follow-up Templates

### assumption (검증 안 된 믿음)

목표: 가장 저렴한 검증 방법을 찾아내기.

```
질문 1 (출처 추적):
  "이 전제를 처음 믿게 된 계기가 무엇인가요?
   누군가 말해준 건지, 직접 관찰한 건지, 추측인지?"

질문 2 (최소 검증):
  "이 가정을 검증하지 않은 채 코드를 작성한다면,
   가장 먼저 깨질 시나리오는 무엇인가요?"

후속 분기:
  - 출처가 나오면 → heuristic으로 승격 재평가
  - "그냥 그럴 것 같아서" → "이 가정 없이도 아이디어가 성립하는가?" 로 전환
```

### heuristic (간접 근거)

목표: 현재 맥락에서도 유효한지 확인.

```
질문 1 (맥락 전이):
  "그 경험/사례가 지금 상황에도 적용된다고 보는 이유는?
   당시와 지금의 가장 큰 차이점은?"

질문 2 (반례 탐색):
  "이 패턴이 작동하지 않았던 사례를 알고 있나요?
   그 경우와 지금의 차이는 무엇인가요?"

후속 분기:
  - 맥락 차이가 크면 → "그 차이가 결과를 바꿀 수 있는가?" 추가 질문
  - 반례가 없으면 → "반례를 찾으려면 어디를 봐야 하는가?"
```

### data-backed (직접 측정)

목표: 데이터의 신뢰도와 유효 범위를 확인.

```
질문 1 (유효 범위):
  "이 데이터가 측정된 조건과 지금 조건이 같은가?
   표본 크기, 시점, 대상이 현재와 어떻게 다른가?"

질문 2 (해석 검증):
  "이 데이터에서 다른 해석도 가능한가?
   같은 수치를 보고 반대 결론을 내릴 수 있는 사람이 있을까?"

후속 분기:
  - 조건 차이가 크면 → "그 차이를 보정하면 결론이 바뀌는가?"
  - 해석이 유일하면 → 등급 유지, 다음 premise로 이동
```

## Domain-Specific Question Banks

### 기술 (Technology)

```
- "기존 시스템과의 통합 포인트에서 가장 큰 기술적 위험은?"
- "이 기술 스택을 선택한 이유가 제약인가, 선호인가?"
- "6개월 후 이 기술이 deprecated될 가능성은?"
- "비슷한 규모의 시스템에서 이 접근이 작동한 사례가 있는가?"
```

### 비즈니스 (Business)

```
- "이 문제를 해결하지 않으면 누가 얼마나 손해를 보는가? (구체적 수치)"
- "경쟁자가 이미 이 문제를 풀었다면, 왜 우리가 다시 풀어야 하는가?"
- "이 기능의 예상 사용 빈도는? 일 1회인가, 월 1회인가?"
- "수익/비용에 직접 영향을 주는 경로가 있는가?"
```

### 시장 (Market)

```
- "타겟 사용자가 현재 이 문제를 어떻게 해결하고 있는가?"
- "이 시장의 크기를 어떻게 측정했는가? 출처는?"
- "얼리 어답터와 메인스트림 사용자의 니즈 차이는?"
- "시장 진입 타이밍이 지금이어야 하는 이유는?"
```

### Generic Fallback

도메인 감지 confidence가 낮을 때 사용:

```
- "이 아이디어의 핵심 가치는 한 문장으로 무엇인가?"
- "이것이 실패한다면 가장 가능성 높은 원인 하나는?"
- "코드를 쓰지 않고 이 가설을 검증할 방법은?"
```
