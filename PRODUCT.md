# Product

## Register

product

## Users

xm 마켓플레이스를 설치해 쓰는 외부 개발자(팀/공개 배포 대상)와 저장소 소유자. 두 가지 소비 모드가 공존한다:
- **사람**: 터미널에서 `xm config`, `xm panel` 등 CLI를 직접 실행. 설정 구조(3-tier scope, 27키 레지스트리)를 모르는 신규 사용자도 위저드만으로 안전하게 설정을 마칠 수 있어야 한다.
- **에이전트**: Claude Code 서브커맨드/파이프로 CLI를 구동(`--json`, `set/get/show`). 결정적 출력과 명확한 exit code가 계약이다.

## Product Purpose

xm은 구조화된 multi-agent orchestration 플러그인 킷이다. config CLI는 그 관제 입구 — 모델 라우팅, 예산, 페이즈 게이트, worktree 병렬 실행 정책을 조정한다. 성공은 "문서를 안 읽은 사용자가 위저드만으로 의도한 tier에 올바른 값을 저장하고, 그 결과를 신뢰하는 것".

## Brand Personality

차분·정밀·안내형 (calm, precise, guided). 조용한 확신 — 화려함 없이 단계마다 현재 상태를 먼저 보여주고, 다음 행동을 안내한다.

## Anti-references

- **이모지 과다**: ✅❌🔍📋⚙️ 난발형 출력. 글리프(◇ ◆ ● ○ ✓ ✗)와 색으로 대체하고 이모지는 쓰지 않는 방향.
- **번호 메뉴 텔레타이프 위저드**: "1) … 2) … 선택: " 식 숫자 입력 메뉴. 화살표 select로 교체 (non-TTY/테스트 폴백은 라인 모드 유지).
- 과장 배너, ASCII 아트 로고, 무의미한 스피너.

## Design Principles

1. **상태를 먼저, 질문은 나중** — 모든 프롬프트는 현재 effective 값과 출처 tier를 보여준 뒤 묻는다. 침묵 쓰기 금지.
2. **터미널도 디자인 시스템이다** — 색 역할·글리프·정렬·간격을 토큰으로 관리(DESIGN.md). 화면마다 같은 어휘.
3. **두 소비자, 두 모드** — 사람은 인터랙티브 위저드, 에이전트/파이프는 결정적 서브커맨드. 한쪽을 위해 다른 쪽을 훼손하지 않는다.
4. **익숙함이 발명을 이긴다** — clack 계열 프롬프트 어휘(◇ 섹션, ● 라디오, │ 레일)를 따른다. 표준 어포던스를 재발명하지 않는다.
5. **실패는 보이게** — 검증 경고·가드·에러는 사용자-가시적으로 (L6). exit code는 정직하게.

## Accessibility & Inclusion

- `NO_COLOR` 존중 (x-panel 선례 유지·확대). 색만으로 의미 전달 금지 — 글리프가 항상 동반.
- 비-UTF8/dumb 터미널용 ASCII 폴백 글리프 세트 (`*`, `o`, `>`, `x`, `!`).
- non-TTY에서 인터랙티브 진입 차단 + 대안 안내 (기구현). 스크린리더 친화: 한 줄에 하나의 의미 단위.
